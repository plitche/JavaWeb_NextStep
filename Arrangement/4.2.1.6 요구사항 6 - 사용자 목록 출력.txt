	4.2.1.6 요구사항 6 - 사용자 목록 출력
		* HTTP는 기본적으로 무상태 프로토콜이라 각 요청간에 상태 데이터를 공유하지 못한다.
		  이 한계를 극복할 수 있는 방법으로 쿠키를 사용한다.
		  HTTP에서 각 요청간의 상태를 공유할 수 있는 유일한 방법이다.
		  
		  하지만 쿠키정보는 클라이언트에 저장해 관리하기 때문에 보안 이슈가 있다.
		  이같은 단점을 보완하기 위해 세션이 등장했다.
		  
		* 웹 브라우저(/user/login) -> HTTP 웹 서버(로그인 성공, Set-Cookie 헤더 값 추가)
		  HTTP 웹 서버(HTTP 응답 + Set-Cookie) -> 웹 브라우저
		  웹 브라우저(/user/list + Cookie) -> HTTP 웹 서버(Cookie 필드 값을 통해 로그인 유무 판단)
		  HTTP 웹 서버(사용자 목록 응답) -> 웹 브라우저
